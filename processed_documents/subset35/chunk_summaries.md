- **Scope and Purpose**: FFFIS stands for 'Form Fit Functional Interface Specification' covering all protocol levels including connector and physical [¶63]. Lower boundary is PROFIBUS FDL layer [¶64]. Upper boundary describes functions between ERTMS/ETCS on-board and STM [¶65]. Objective: enable ERTMS/ETCS on-board to connect to any STM (interchangeability) [¶66]. Functionality in level NTC/mode SN equals legacy National Train Control system [¶67]. Transitions between ERTMS/ETCS and National Systems, and between National Systems, are seamless without additional trackside constraints beyond Eurobalises for level transitions [¶68]. Access to standardised interfaces (DMI, Train Interface, Juridical Recording, odometer) minimises number of interfaces for multiple National Systems [¶70]. Use of specific interfaces by National Systems permitted if no additional requirements on ERTMS/ETCS on-board; otherwise non-compliant [¶71–¶72]. Interface 'K' (KER balise) is not part of FFFIS STM as data not over PROFIBUS [¶73].

- **General Requirements**: STM identified by unique NID_STM equal to NID_NTC values from document [11] [¶76]. STM uses common Time from ERTMS/ETCS on-board [¶77]. Only one STM active (supervising) at a time [¶78]. ERTMS/ETCS on-board monitors STM interface safety integrity and applies emergency brake on active STM failure; non-active STM failure not critical [¶79–¶80]. Isolation of STM possible without bus disturbance [¶82].

- **Functional Architecture**: ERTMS/ETCS on-board allows STM to communicate with: DMI, STM Control, Reference Time, BIU, TIU, Juridical Data, Odometer (Figure 1) [¶85–¶93]. Data transmitted via Multicast or Point-to-Point connections [¶96].

- **On-board Functions and Data**:
    - *Reference Time*: ERTMS/ETCS on-board provides common reference time to all STMs per [2] [¶97–¶98].
    - *Odometer*: Data/parameters sent via multicast to all STMs [¶99–¶100].
    - *Train Interface (TIU)*: Subset of train interface signals from [6] transmitted as Train Interface FFFIS STM signals [¶101–¶102]. Command signals: Regenerative Brake, Magnetic Shoe Brake, Eddy Current Brake (Service/Emergency), Pantograph, Air Tightness, Main Switch/Circuit Breaker, Traction Cut Off [¶106, ¶110]. Status signals: Traction status, Direction Controller information, Cab Status [¶108, ¶111]. Service/Emergency Brake commands/status are in BIU [¶107, ¶109].
    - *Brake Interface (BIU)*: Part of Train Interface with brake interface parameters, command, status/availability of Emergency and Service Brake access [¶112–¶113]. Physically separated for different safety/performance levels [¶114]. Brake status gives availability [¶115].
    - *Juridical Data*: STM transmits national juridical data forwarded to On-Board Recording Device [¶116–¶117].
    - *STM Control Function*: Controls STM state and version compatibility; handles ETCS data, Specific NTC Data Entry/View, ETCS status data, language, test procedure, override, airgap data, STM max speed/system speed/distance, bus address/ safety level/availability, STM failure status, active Interface 'K' Antenna/BTM, BTM alarm data [¶118–¶130].
    - *DMI*: Allows active STM to dialogue with driver for default window: management of buttons, indicators, sounds, text messages, supervision information [¶131–¶138].

- **Function Access per Mode/Level**: Table specifies access for STM Control, Reference Time, DMI, Juridical Data, Odometer, TIU command/status, BIU command/status across modes NP, SB, PS, SH, FS, LS, SR, OS, SL, NL, UN, TR, PT, SF, SN, RV [¶140–¶146]. 'x' = allowed in all Levels, '(x)' = allowed if possible, 's' = only for active STM, 'h' = for HS preliminary DMI requests [¶140–¶144]. On function failure, it isolates from bus and tries to close connection [¶145].

- **PROFIBUS Bus**: Interface bus per [5] up to FDL layer [¶149–¶150]. Configuration: Baud Rate 1500 Kbps, min TSDR 11 tBit, max TSDR 150 tBit, TSL 300 tBit, TQUI 0 tBit, TSET 1 tBit, TTR 30000 tBit (20 ms), G 10, HSA 126, max_retry_limit 1 [¶152–¶162]. Allows max 200 m per segment, 32 stations on cable type A; repeaters for greater length/ more stations [¶163]. PROFIBUS may have other communications [¶164].

- **Physical Connection**: Default medium RS-485 twisted pair shielded copper cable [¶165–¶166]. Connectors: 9-pin female D-SUB per PROFIBUS specs [¶167].

- **Bus Redundancy**: Retransmission per [3] [¶168–¶169]. STM and ERTMS/ETCS on-board have at least one bus interface, may have two [¶170]. If different number of buses, only one connected [¶171]. Dual bus managed by 'Redundancy Supervisor' per [3] [¶172].

- **Safety**: Three safety protocol levels: SL4, SL2, SL0 [¶174–¶177]. Equipment shall not implement SL higher than its SIL [¶180]. ERTMS/ETCS on-board functions implement all safety protocols up to SL corresponding to function SIL [¶181]. Justification per [8] [¶178].

- **Physical Addressing**: Each STM has one physical address [¶183]. ERTMS/ETCS on-board may use one or several [¶184]. STM handles one address per function [¶185]. Multiple STMs sharing same address must establish own application layer connections with different NID_STMs [¶186]. Allocation: Address 2 for STM Control Function; 0,1,2,3…19 for other functions; 20…49 unused; 50…69 configurable STM range; 70…126 for STMs (NID_NTC+70); 127 reserved for Broadcast/Multicast [¶190]. Default STM physical address = NID_NTC + 70 [¶191]. Configurable range for STMs where NID_STM+70 exceeds PROFIBUS range [¶192]. Configurable to resolve conflicts [¶194].

- **Function Addressing**: SAPs support communication between STMs and functions [¶196–¶197]. All functions have fixed SAP regardless of physical address [¶198–¶199]. SSAP and DSAP have same value [¶200]. SAP table: DMI channels 3,4,1,2 (SAP 000000, 000001, 000100, 000101, point-to-point), Juridical Data (000010, point-to-point), Reserved (000011, 000110, 000111), Unused (001XXX (8), 01XXXX (16)), Reference Time (100000, multicast), STM Control (100001, point-to-point), Train Interface (100101, point-to-point), Brake Interface (100110, point-to-point), Odometer (100111, multicast for version X=4), Broadcast (111111) [¶202–¶203]. Only one source for Reference Clock SAP and Odometer SAP [¶204–¶205].

- **Protocol Layers**: Application Layer [4], Safe Time Layer [2], Safe Link Layer [3], PROFIBUS FDL layer [5] (Figure 2, 3) [¶207, ¶209, ¶211]. Safe Time and Safe Link together are Safety Layers [¶208].

- **Connection Management**: Connection established when version check successful [¶215]. STM initiates connection [¶216]. After 2 failures, retry after 10 seconds [¶217]. Version check: STM sends 'FFFIS STM version number' to function with state report [¶219]. Function checks: if version lower than lowest supported → close connection; if among supported → send highest supported with same X; if higher than highest → close [¶221–¶223]. STM checks version from function; if compatible → transmit data; if not → close connection [¶224–¶225]. Closing via Safety Layers [¶228].

- **Multicast Connection**: Sender opens separate connection for each 'FFFIS STM version number' in legal backward compatibility envelope [¶233]. Table 6.5.1.5 contains one SAP per version number for Odometer [¶234]. Sender transmits corresponding version number repeatedly for restarting STMs [¶235]. STM checks compatibility; ignores data from incompatible multicast [¶236–¶237].

- **STM States**:
    - *No Power (NP)*: Unpowered [¶239–¶240].
    - *Power On (PO)*: Default after switch on; synchronise Safe Time Layer; establish connection with STM Control Function; send 'Specific NTC Data Need' [¶241–¶245]. After ERTMS/ETCS on-board sends bus addresses/safety levels, STM may establish further connections [¶246]. After ETCS status data sent, STM may request CO state [¶247].
    - *Configuration (CO)*: Wait for exchange of configuration data: ETCS data (10.4), TIU status/availability, BIU status/availability, odometer performance parameters, brake performance parameters (max time delay for processing brake commands) [¶248–¶255]. Note: some data may be sent earlier [¶256]. Brake performance used in braking curves [¶257]. If no Specific NTC Data needed → request CS; if in NL/SL mode → request CS even without all ETCS data; if data needed → request DE [¶258–¶261]. May close connections except STM Control when exiting CO [¶262].
    - *Data Entry (DE)*: For STM requiring Specific NTC Data for operation; entered only once at start-up [¶263–¶265]. Perform Specific NTC Data Entry procedure (10.7) [¶266]. When terminated → request CS (even if data not received/skipped/ invalid) [¶267–¶268].
    - *Cold Standby (CS)*: Initialised, tested, configured, has all required info but reception off [¶269–¶270]. Exception: Specific NTC Data may be invalid [¶271].
    - *Hot Standby (HS)*: Processes trackside information to be ready for supervision [¶272–¶273]. Can send 'STM max speed' (V_STMMAX) for smooth transition [¶276–¶277]. Can send 'STM system speed' (V_STMSYS) and 'STM system distance' (D_STMSYS) for national trackside detection [¶278–¶279]. May close connections except STM Control when ordered to CS [¶280].
    - *Data Available (DA)*: Responsible for train movement supervision per national trackside info [¶281–¶282]. May close connections except STM Control when ordered to CS [¶283].
    - *Failure (FA)*: Unable to work; no messages except state report [¶284–¶286].

- **STM Manager System (STM Requirements)**:
    - *Scope*: Defines STM state handling [¶288–¶290].
    - *Transitions Table*: Conditions for transitions between NP, PO, CO, DE, CS, HS, DA, FA defined [¶291, ¶295–¶297]. Conditions include ETCS orders (Configuration, Data Entry, Cold Standby unconditional/conditional, Hot Standby, Data Available, Failure) and STM decisions [¶296–¶297]. Note: STM in DA in National Trip Procedure sends 'National Trip Procedure' cyclically to fulfil timeout; if mode changes to TR, expected to enter CS even if National Trip Procedure not finished [¶298].
    - *General Requirements*: STM antenna only energises trackside in HS/DA or for test [¶300–¶302]. Invalid state order → FA [¶303]. STM reports NID_STM and current state on all point-to-point connections with each message and on state change [¶304–¶308]. FA reported if possible [¶310].

- **STM Control Function**:
    - *General*: Configurable list of STMs [¶312–¶313]. Maintains list of available STMs (CS, HS, DA) [¶314]. Level NTC X 'Available for use' if STM X available [¶315]. On connection: send available functions, bus addresses, safety levels [¶316–¶319]. Only Juridical Data and DMI channels 2,3,4 may be marked unavailable [¶320]. Inform about active DMI channel on change or connection [¶321–¶323].
    - *Association STM X to Level NTC X*: Look-up table maps NID_NTC to one or more NID_STM with priority [¶325]. If National System encapsulates others, table needed for mapping [¶326]. 'STM X' means STM associated to Level NTC X [¶327]. Association rules: on level transition order, if ETCS system version <2.0 or NID_NTC not in table → STM with NID_STM=NID_NTC; otherwise highest priority available STM linked [¶328–¶332]. For airgap data → same as level transition [¶333]. On driver selection/validation → similar logic; if no available, then connected non-failed non-isolated STMs [¶334–¶336]. Association kept until level left or Stand-By/No Power mode [¶337–¶338].
    - *STM Manager System (State Orders)*: Manages states of connected STMs [¶339–¶341]. State order table lists events leading to orders (Figure 10.3.2.2) [¶343–¶345]. Conditions for orders detailed in table [¶347–¶352]. STM Control Function sends order when conditions valid [¶354]. Exceptions: no order for NP/PO; no FA order if STM already reported FA [¶355–¶356]. Conditional CS order for transitions A4b and B4b; unconditional for others [¶357]. Note about Q16 and Trip mode [¶358].
    - *State Order/Report Requirements*: STM Control Function does not evaluate conditions (except FA) until STM reports requested state [¶360–¶361]. Active STM defined from DA order sent until next order or different state report [¶362]. Emergency brake commanded when conditional CS sent to STM in National Trip Procedure until CS or standstill [¶363–¶364]. Emergency brake applied when level NTC X, mode SN, STM X installed but not available (exception: if isolated via Train Interface) [¶365–¶366]. Release conditions: reconnection with DA state, level change to 0/1/2/3, level change to NTC Y not associated, mode SN left, isolation input [¶367–¶372]. Reconnection accepted except for FA/PO after final disconnection [¶373]. Driver informed about STM X unavailability when level NTC X, mode SN or NL≥5s, STM not available, not isolated; 5s delay for NL due to CS request [¶374–¶380].
    - *ETCS Data*: Includes subset of ETCS Train Data (categories, length, traction/brake parameters, max speed, loading gauge, axle load, traction systems, airtight system) [¶381–¶390]. Includes subset of Train Data entry fields (Train Type) [¶391–¶392]. Extra data handled in Specific NTC Data Entry [¶393]. Traction/brake parameters include times, brake position, brake percentage [¶394–¶399]. Includes subset of Additional Data (Train Running Number, ETCS identity, adhesion factor, Date/Time UTC) [¶400–¶405]. Includes National/Default Values per [1] [¶406]. Transmit valid Train Data when validated [¶407–¶408]. Transmit Additional Data on CO state and on change (except date/time) [¶409–¶411]. Transmit National/Default Values on CO and on change [¶412–¶414].
    - *ETCS Status Data*: Current mode and level per [1]; sent to all connected STMs on change and to STM on connection [¶415–¶418].
    - *Language*: Transmitted to all on change and to STM on connection [¶419–¶422].
    - *Specific NTC Data Entry*: National data requested from driver [¶424–¶425]. STM may use ETCS data to reduce entry [¶426]. Each data has unique identity (NID_STM + Data Identifier) [¶427]. Process is 'Specific NTC Data Entry' [¶429]. Possible at start-up and during mission [¶430]. Responsibilities: ERTMS/ETCS on-board handles dialogue, range checks (if configured), transmission after validation [¶431–¶432]. STM checks content and may be exempted if configured [¶433]. Driver may skip entry for a STM [¶435]. STM may request data via 'Specific NTC Data Need' if data invalid [¶436]. Invalidity reasons: entry not performed/aborted/skipped, data change from non-driver sources, STM internal functions [¶437–¶442]. On receiving need in FS, LS, SR, OS, UN, TR, PT, SN → inform driver [¶443]. Delete driver info on Train Data entry initiation, STM failure, or isolation [¶444]. Request includes label, optional default, optional dedicated keyboard values [¶446]. Unless dedicated keyboard, alphanumeric keyboard used [¶447]. Configurable parameters: window titles, keyboard type (numeric/enhanced numeric/ alphanumeric), leading zeros, min/max values for range check [¶448–¶453]. Brake command/release rules per [1] apply for NTC data modification/revalidation [¶454].
    - *Entry Procedure*: When ETCS Train Data validated and STM in CO/DE/CS/HS/DA → send START flag and ETCS Train Data [¶456–¶457]. During procedure, send STOP flag on 'End of Specific NTC Data Entry', timeout (10s), abortion, or skip [¶458–¶462]. Note about abortion reasons [¶463]. ETCS Train Data also sent without flags outside entry [¶465]. On receiving ETCS data while entry ongoing: if data needed → send 'Specific NTC Data Entry request'; if not → send 'End of Specific NTC Data Entry' [¶466–¶468]. After receiving request, perform exchanges with driver [¶469]. On validation by driver → send Specific NTC Data [¶470]. STM checks data; if OK → 'End'; if not → send request again [¶471–¶473]. Supervise separate timeout 10s for each STM from ETCS Train Data or Specific NTC Data until response [¶474–¶476].
    - *Data View*: Allow driver to view Specific NTC Data View values [¶482–¶483]. On trigger → send Request to all available STMs [¶484]. STM responds with values or 'No values' [¶485–§87]. Display values to driver [¶488]. Supervise separate 10s timeout [¶489].
    - *Test Procedure*: STM can send Test Procedure Permission Request with Test Identity [¶490–¶491]. ERTMS/ETCS on-board grants when suitable [¶492–§93]. STM performs test and reports End with result/ text message [¶494–¶495].
    - *Override*: Provides override for active system and system to be activated without brakes [¶497–§98]. When Override activated in active system (ETCS or STM), all systems notified; each monitors specific limits [¶499]. After level transition, activated system can have Override immediately [¶500]. ETCS Override activated when in Level NTC and active STM reports activation [¶501–§02]. Report Override status to STMs on change and on connection [¶504–§06]. In SN mode, no speed supervision by ETCS except for active STM [¶508].
    - *Airgap Messages*: Data from RBC or Balise Group (non-infill) transmitted to STM associated to Level NTC in data [¶509–§10]. Add odometer reading of balise group or LRBG [¶512–§13].
    - *Max Speed / System Speed*: V_STMMAX from HS STM included in MRSP from level transition border [¶514–§16]. New value replaces previous [§517]. If STM not available → V_STMMAX=0 [§518–§19]. V_STMSYS and D_STMSYS included in MRSP as speed restriction from location in rear of border [§520]. New values replace previous [§521]. On deletion of announcement → delete V_STMSYS and V_STMMAX, stop supervision [§523–§25]. Note about new transition replacing previous [§526]. After border → supervise V_STMMAX as ceiling speed until STM DA report [§529]. If STM in FA after border → stop supervision [§530]. If level changes → delete V_STMMAX and stop [§531].
    - *National Trip Procedure Validity*: Active if packet received within last 10 seconds per [1] [§532–§33]. Note about 10s after release [§534].
    - *Failure Status Display*: On FA report/order → inform driver [§535–§36]. At Start of Mission after Train Data validation, if STM known as installed not in CO/CS/HS/DA and not isolated → inform driver [§537–§38].
    - *Interface 'K'*: If using alternative 1 of [10], indicate to KER STMs if can guarantee data from intended Antenna/BTM [§539–§40]. If not, provide active Antenna/BTM info to KER STMs on change and on connection [§541–§43]. Enables supervision per [10] [§544].
    - *BTM Alarm Data*: Send BTM alarm status and whether antenna within announced Big Metal Mass track condition to all STMs on status change and on connection [§545–§48]. Sent regardless of alarms being ignored per [1] [§549].

- **TIU and BIU Functions**:
    - TIU transmits train interface inputs status/availability to STM on change and on connection; also command configuration [§551–§54].
    - BIU transmits brake performance parameters on connection; brake status/availability on change and on connection [§555–§58].
    - When STM commands service brake, indicate if backup emergency brake needed [§559–§60].

- **Odometer Function**:
    - Transmits odometer info regularly: estimated distance, direction, estimated speed, confidence intervals (min/max distance, min/max speed) [§561–§64].
    - Time stamped with Reference Time from Safe Time Layer [§565–§66].
    - Positive direction = forward (cab A), positive speed/increasing distance; negative = backward, negative speed/decreasing [§567–§68]. Cab allocation is implementation issue [§569].
    - Odometer not reset while powered on [§570–§71].
    - Configuration data transmitted [§572].
    - Speed definitions: V_Est (estimated as in [1]), V_Max (most positive including under-reading for positive movement, lowest absolute for negative), V_Min (most negative including over-reading for positive, highest absolute for negative) [§574–§78]. Figure 9 example [§579].
    - Distance definitions: D_Est (most probable position), D_Max (most positive position with all over/under-reading accumulated), D_Min (most negative position with all over/under-reading accumulated) [§581–§84]. Confidence interval per [7] [§585]. Resolution parameter in each report [§586]. STM can compute max/min travelled distances using resolution and D_Max/D_Min [§587–§89]. Distance parameters may wrap individually [§590].
    - Configuration info: T_OdoCycle (typical cycle time), T_OdoMaxProd (maximum production delay including clock synchronisation) [§591–§96]. Figure 10 [§598].

- **DMI Function**:
    - Allows driver interaction with ERTMS/ETCS on-board for national info in default window [§600–§01].
    - Provides unified and customisable DMI services [§602]. STM using customisable provides configuration data stored in ERTMS/ETCS on-board [§603]. STM using unified provides no configuration data [§604]. Unconfigurable parts handled same way [§605].
    - *General*: Only active STM communicates with driver [§607]. On loss of active status → delete all DMI objects [§608]. On DMI connection disconnection → delete objects after 2s timeout [§609–§10]. If re-established within 2s → delete on re-establishment [§611].
    - *Preliminary Requests*: ERTMS/ETCS on-board receives and stores from STM in HS, displays on DA report [§613–§14]. On PO/CS/FA report → delete preliminary requests [§615].
    - *Speed Display*: If configured not to display in SN mode → inhibit only after DA report [§616].
    - *DMI Channels*: Up to 4 channels, each one connection [§617–§18]. At most one active; only active used for DMI objects at application level [§619]. Inactive channels may remain open [§620]. ERTMS/ETCS on-board may report active channel even without driver interaction (e.g., no cab active) [§621]. On active channel change → delete all DMI objects of STM [§622].
    - *DMI Objects*:
        - *Identities*: Indicators and buttons have unique identity (NID_STM + Indicator/ Button Identifier) [§624–§25]. NID_STM provided implicitly [§626]. Identifier used for state change, move/remove, event transmission; also for properties in customisable service [§627–§28]. Icons have identity (NID_STM + Icon Identifier) [§630–§31]. Sounds have identity (NID_STM + Sound Identifier) [§632–§33]. Position Identifiers specify position [§634]. For unified service → area of ETCS layout per [9]; for customisable → cell coordinates and size [§635–§36].
        - *Text Messages*: Request includes Text Identifier, string, display attribute, optional acknowledgement request [§637–§38]. On acknowledgement → report to STM [§639]. Delete on request with Text Identifier [§640–§41]. If new request with same identifier → replace [§642]. Display attribute specifies colour, background, flashing mode, group [§643]. Flashing mode: slow/fast/none, normal/ counterphase [§644].
        - *Indicators*: Display without input; request: Indicator Identifier, optional Icon Identifier, optional caption, Position Identifier, display attribute [§646–§52]. Icon used in customisable service to retrieve icon [§653]. Display attribute specifies background colour, flashing mode, caption colour [§654]. Flashing mode: slow/fast/none, normal/ counterphase [§655].
        - *Buttons*: Functional extension of Indicators [§656–§57]. DMI transmits push and release events to STM with timestamp using Reference Time [§658–§60].
        - *Sounds*: Request: optional Sound Identifier; for unified DMI: sequence of segments (duration, frequency); indication if continuous/one-shot/stop [§661–§66]. Sound Identifier used in customisable service to retrieve sound; in both services to stop sound [§667–§68]. DMI manages two simultaneous STM sound requests [§669–§70].
        - *Supervision Information*: Two sets: speed/distance values (Permitted Speed, Target Speed, Target Distance, Release Speed, Intervention Speed) and colours/display modes (current train speed pointer, colours for speeds and modes, display modes for each element) [§671–§98]. DMI uses last received values [§699].
    - *Customisable DMI Service Configuration*: Data for each STM: NID_STM, list of Indicators (id, font size, horizontal/vertical alignment), two lists of Indicator positions (for soft key and touch screen: id, x:y offset, size), list of Buttons (id, font size, alignment), two lists of Button positions (id, offset, size, linked soft key), list of Icons (id, bitmap, display text upon icon), ETCS speed/ distance supervision (Yes/No, speed dial range), flashing options (slow/fast frequency, style: yellow frame/whole area), list of Sounds (id, WAV file), list of moved ETCS layout areas (A4, B7, C1, C7-C9, E1, F1-F5, G13: new x:y offset, new soft key for buttons) [§700–§51]. Recapping table with multiplicities and ranges [§750–§53].

- **Juridical Data Function**:
    - ERTMS/ETCS on-board receives and forwards national juridical data to On-Board Recording Device [§754–§55].
    - STM uses Reference Time for timestamping; timestamp represents event occurrence time for chronology [§756–§58].

- **Limitations**:
